Unidad 1 – Reconocimiento de Funciones

Descripción mejorada. Se desarrolla un programa que, a partir de una función matemática ingresada por el usuario con un máximo de dos variables, identifica dichas variables y cuantifica el número de operaciones presentes. Se consideran sumas, restas, multiplicaciones, divisiones, potencias y multiplicaciones implícitas (como 5x o xy), evitando confundir nombres de funciones (sin, log, sqrt, etc.) con variables. El resultado presenta: variables detectadas, número de variables distintas y total de operaciones encontradas.

Objetivos de aprendizaje

  • Diferenciar símbolos de función y variables.
  • Detectar y contar operadores explícitos e implícitos.
  • Resumir métricas de complejidad de la expresión.

Entradas y salidas

  • Entrada: cadena con la expresión (hasta 2 variables).
  • Salida: listado de variables, conteos por operador y total.

import tkinter as tk
from tkinter import ttk, messagebox

# ===============================
# Función principal de análisis
# ===============================
def analizar(event=None):
    expresion = entrada.get()

    if not expresion.strip():
        messagebox.showwarning("Error", "Por favor ingresa una función matemática.")
        return

    variables = set()
    operaciones = 0

    for i, c in enumerate(expresion):
        if c.isalpha():
            variables.add(c)
        elif c in "+-*/^":
            operaciones += 1

        # Detección de multiplicación implícita
        if i < len(expresion) - 1:
            siguiente = expresion[i + 1]
            if c.isdigit() and siguiente.isalpha():
                operaciones += 1
            elif c.isalpha() and siguiente.isalpha():
                operaciones += 1

    resultado_texto = (
        f"📡 Análisis completado:\n\n"
        f"🔸 Variables encontradas: {', '.join(sorted(variables)) if variables else 'Ninguna'}\n"
        f"🔸 Número de variables: {len(variables)}\n"
        f"🔸 Número de operaciones: {operaciones}"
    )

    resultado.configure(state="normal")
    resultado.delete("1.0", "end")
    resultado.insert("1.0", resultado_texto)
    resultado.configure(state="disabled")


# ===============================
# Función para limpiar la entrada

def limpiar():
    entrada.delete(0, "end")
    resultado.configure(state="normal")
    resultado.delete("1.0", "end")
    resultado.configure(state="disabled")

Unidad 1 – Graficar Funciones Lineales (ASCII)

Descripción mejorada. Una función es una relación entre dos conjuntos que asocia a cada entrada x un único valor de salida y, denotado y = f(x). Las funciones lineales se modelan como f(x) = m·x + b, donde m es la pendiente (inclinación de la recta) y b la ordenada al origen (punto de corte con el eje y).

Se implementa un programa en Python —sin librerías gráficas— capaz de reconocer dos expresiones lineales y generar un gráfico ASCII en un plano cartesiano con rango estándar de ±20 unidades. El resultado muestra ejes, origen e intersección aproximada, facilitando la interpretación de pendientes y cortes.

import tkinter as tk
from tkinter import ttk, messagebox

# ===============================
# Función para preparar expresiones lineales
# ===============================
def preparar_expresion(expr: str) -> str:
    expr = expr.replace(" ", "")          # quitar espacios
    expr = expr.replace("^", "**")        # potencia
    expr = expr.replace("-x", "-1*x")     # caso -x
    expr = expr.replace("+x", "+1*x")     # caso +x
    if expr.startswith("x"):              # si empieza con x
        expr = "1*" + expr
    expr = expr.replace("x", "*x")        # poner multiplicación
    expr = expr.replace("**x", "*x")      # corregir si se duplicó
    return expr

# ===============================
# Función para graficar las funciones en ASCII
# ===============================
def graficar():
    f1 = entrada1.get()
    f2 = entrada2.get()

    if not f1.strip() or not f2.strip():
        messagebox.showwarning("Error", "Por favor ingresa ambas funciones.")
        return

    func1 = preparar_expresion(f1)
    func2 = preparar_expresion(f2)

    xmin, xmax = -20, 20
    ymin, ymax = -20, 20

    resultado_texto = ""

    for y in range(ymax, ymin - 1, -1):
        linea = ""
        for x in range(xmin, xmax + 1):
            try:
                y1 = eval(func1)
            except:
                y1 = None
            try:
                y2 = eval(func2)
            except:
                y2 = None

            cond1 = (y1 is not None and abs(y - y1) < 0.5)
            cond2 = (y2 is not None and abs(y - y2) < 0.5)

            if cond1 and cond2:
                linea += "#"
            elif cond1:
                linea += "*"
            elif cond2:
                linea += "o"
            elif x == 0 and y == 0:
                linea += "+"
            elif x == 0:
                linea += "|"
            elif y == 0:
                linea += "-"
            else:
                linea += " "
        resultado_texto += linea + "\n"

    # Leyenda
    resultado_texto += "\nLeyenda del gráfico:\n"
    resultado_texto += "  * = Función 1\n"
    resultado_texto += "  o = Función 2\n"
    resultado_texto += "  # = Intersección\n"
    resultado_texto += "  | = Eje Y\n"
    resultado_texto += "  - = Eje X\n"
    resultado_texto += "  + = Origen (0,0)\n"

    resultado.configure(state="normal")
    resultado.delete("1.0", "end")
    resultado.insert("1.0", resultado_texto)
    resultado.configure(state="disabled")

Unidad 1 – Método de Regula Falsi

Descripción mejorada. El método de la Regula Falsi (o falsa posición) es un procedimiento por intervalos para hallar raíces reales de f(x)=0. Parte de un tramo [a,b] tal que f(a)·f(b)<0, lo que asegura un cambio de signo y la existencia de al menos una raíz. En cada iteración se traza la secante entre (a,f(a)) y (b,f(b)) y se toma su intersección con el eje x como aproximación; luego se actualiza el extremo que preserva el cambio de signo, manteniendo acotada la solución.

El método combina la robustez del enfoque por bisección (la raíz nunca escapa del intervalo) con una direccionalidad propia de la secante, que suele acelerar el acercamiento a la raíz. En casos de estancamiento por curvatura, la variante Illinois atenúa el peso de un extremo para recuperar progreso. Los criterios de parada habituales son una tolerancia en |f(x)| y/o en la longitud de [a,b], además de un máximo de iteraciones.

Unidad 1 – Método de Newton-Raphson

Descripción mejorada. El método de Newton–Raphson aproxima raíces reales de funciones continuas y derivables mediante la actualización xₙ₊₁ = xₙ − f(xₙ)/f’(xₙ). Se interpreta como la intersección de la recta tangente en xₙ con el eje x. Con un punto inicial adecuado y funciones suaves, su convergencia suele ser cuadrática, es decir, muy rápida.

Aplicación en Python. Primero se grafica f(x) en un intervalo definido por el usuario para identificar visualmente zonas con posibles raíces. Si se confirma, se solicita un valor inicial x₁ y se ejecuta Newton–Raphson con derivada numérica central, controles ante pendientes casi nulas y criterios de parada por tolerancia y por iteraciones máximas. El programa reporta la raíz aproximada y el número de iteraciones, y muestra la traza de los puntos generados.

    def coord_to_pixel(x, y):
